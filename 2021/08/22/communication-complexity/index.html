    通信复杂度

    Posted on 2021-08-22 Edited on 2021-08-28 In notes
    Symbols count in article: 4.5k Reading time ≈ 4 mins.
    这玩意资料真难找

    拿的斯坦福的Tim Roughgarden教授的讲稿看的，基础有限

    这里是一些对于不等式的记录和练习的记录...

    Inequalities

    • Chebyshev's inequality

    设随机变量\(X\)​的期望\(E(X) = \mu\)​，方差\(D(X) = \sigma^2\)​，那么对于任意正数\(\epsilon\)​，有

    \[P( |X - \mu| \geq \epsilon) \leq \frac{\sigma^2}{\epsilon^2}\]

    这里对离散的情况进行证明，连续的情况只需对上下限作出相应的更改

    \(\begin{align}P(|X - \mu| \geq \epsilon) &= \sum_{|x-\mu| \geq \epsilon} p(x) \\ &\leq \frac{1}{\epsilon^2}\sum_{|x-\mu|\geq \epsilon} (x-\mu)^2 p(x) \\&\leq \frac{1}{\epsilon^2}\sum (x-\mu)^2 p(x) = \frac{\sigma^2}{\epsilon^2} \end{align}\)​

    • Markov's inequality

    设随机变量\(X\)的绝对值的期望存在，​那么对于任意正数\(\epsilon\)​，有

    \[P(|X| \geq \epsilon) \leq \frac{1}{\epsilon} E(|X|)\]​​

    还是只对离散情况进行证明

    \(\begin{align}P(|X| \geq \epsilon) &= \sum_{|x| \geq \epsilon} p(x) \\ &\leq \frac{1}{\epsilon}\sum_{|x|\geq \epsilon} |x| p(x) \\&\leq \frac{1}{\epsilon}\sum |x| p(x) = \frac{E(|X|)}{\epsilon} \end{align}\)​

    • Cantelli's inequality

    设随机变量\(X\)​的期望\(E(X) = \mu\)​，方差\(D(X) = \sigma^2\)​，那么对于任意正数\(\epsilon\)​，有

    \[P( X - \mu \geq \epsilon) \leq \frac{\sigma^2}{\epsilon^2 + \sigma^2}\]​

    令\(Y = X - \mu\)​，则\(E[Y] = 0\)​, \(Var[Y] = Var[X - \mu] = \sigma^2\)，那么\(\forall x > 0\),

    \(\begin{align} P(Y \geq \epsilon) &= P(Y + x \geq \epsilon + x) = P(\frac{Y+x}{\epsilon+x} \geq 1) \\ & \leq P((\frac{Y+x}{\epsilon+x})^2 \geq 1) \leq E[(\frac{Y+x}{\epsilon+x})^2)] = \frac{\sigma^2 + x^2}{(\epsilon+x)^2}\end{align}\)

    其中用到了Markov不等式，最后代入\(x = \sigma^2 / \epsilon\)​​​即可

    如果令\(\epsilon = \sigma\)，我们得到\(P(X - \mu \geq \sigma) \leq \frac{1}{2}\)，类似的有\(P(X - \mu \leq -\sigma ) \geq \frac{1}{2}\)，因此中位数\(v\)一定满足\(|v - \mu| \leq \sigma\)

    Protocol

    One way communication

    Alice得到输入\(x\)，Bob得到输入\(y\)，Alice通过计算，得到\(z\)，将\(z\)发送给\(y\)，Bob通过\(z, y\)计算出一个原本需要\(x, y\)计算的函数

    public coins

    对于Alice和Bob而言，在开始通信之前，可以认为它们共有一个无限长的01随机序列

    当然，对于一些问题的讨论，也有可能引入private coins的假设来进行讨论，但这时一般会明确指明

    two-sided error

    不论\(f(x, y) = 0\)，或\(f(x, y) = 1\)，只要计算的结果和原本不一样，就认为计算错误

    （这是为了区别认为当\(f(x, y) = 0\)时计算出\(f^*(x, y) = 1\)的结果不算错误的情况）

    Arbitary constant error probability

    任何错误概率在\((0, \frac{1}{2})\)内的protocol的通信复杂度相同

    worst-case communication

    通信复杂度以最坏情况计算，即如果一个protocol以s的bit作为通信的bit，那么Alice最多发送s的bit给Bob

    Theorem

    (Yao 1983) Let \(D\) be a distribution over the space of inputs \((x, y)\) to a communication problem, and \(\epsilon \in (0, \frac{1}{2})\). Suppose that every deterministic onw-way protocol \(P\) with \[Pr_{(x, y) \sim D} [P \;wrong\;on \;(x, y)] \leq \epsilon\] has communication cost at least \(k\). Then every randomized one-way protocol \(R\) with error at most \(\epsilon\) on every input has communication cost at least \(k\).

    这个定理表明如果要证明一个随机的protocol的复杂度下限为\(c\)，那么我们只需要证明，存在一种概率的分布，在这种分布下，所有的确定性算法的复杂度都不会小于\(c\)即可，事实上，这个定理不仅是充分的，而且是必要的

    Lower Bounds

    数据流，就是一串数据一个一个输入，且只能输入一遍的流...

    F2

    对于一串数据\(x_1, x_2,..., x_n \in [m]\)，计算\(\sum_{i=1}^m f_i^2\)，其中\(f_i\)表示\(i\)出现的次数

    建立\(t\)个\(h\)函数，这些\(h\)函数把\([m] \to \{1, -1\}\)，取\((\sum_{i=1}^n h(x_i))^2\)作为答案

    计算之后，观察到期望和\(F_2\)相等，方差由\(t\)来控制

    F0

    对于一串数据\(x_1, x_2,..., x_n \in [m]\)​，计算\(\sum_{i=1}^m [f_i>0]\)​，其中\(f_i\)​表示\(i\)​出现的次数

    注意到在\([0, 1]\)​之间扔\(k\)​个随机取值的数，那么最小数的期望为\(\frac{1}{1+k}\)，最大数的期望为\(1 - \frac{1}{1+k}\)，由此可以建立一个hash函数，通过最小数的期望来得到\(F_0\)

    Index

    Alice得到\((x_1, x_2, ..., x_n) \in \{0,1\}^n\)，Bob得到\(i \in [n]\)，计算\(f(x, i) = x_i\)

    这个问题是\(\Omega(n)\)​的，由Kermer在1999年解决

    考虑输入是均匀分布，对于每一个\(x\)，Alice会生成一个确定的\(z\)，而对于这个\(z\)，Bob对于每一个\(i\)会生成一个bit作为答案，把这些bit组合到一起，称为\(w\)，那么\(\frac{1}{n}|x \oplus w|\)就是\(x\)错误的概率，然后简单的计算，得出在只用\(cn\)个bit的情况下，全局的错误概率总是\(\geq \frac{1}{8}\)的来得出结果

    Disjointness

    Alice得到\((x_1, x_2, ..., x_n) \in \{0,1\}^n\)，Bob得到\((y_1, y_2, ..., y_n) \in \{0,1\}^n\)，计算\(f(x, y) = [\exists i \in [n], x_i = y_i]\)

    这个问题也是\(\Omega(n)\)的，通过规约到Index来解决

    在Index中，令\(x' = x\)，\(y' = e_i\)，\(e_i\)即只在第\(i\)位为\(1\)的向量，那么对\((x', y')\)，计算Disjointness，就可以得到Index的答案

    Gap-Hamming

    给定参数\(t, \epsilon\)，Alice得到\((x_1, x_2, ..., x_n) \in \{0,1\}^n\)，Bob得到\((y_1, y_2, ..., y_n) \in \{0,1\}^n\)，计算\(f(x, y) = \begin{cases} 0 &(d_H(x, y) > t+\epsilon n) \\ 1 & (d_H(x, y) < t - \epsilon n) \\ undefined & o.w.\end{cases}\)​

    对于一般的\(t\)来说，这个问题还是\(\Omega(n)\)的，通过规约到Index来解决，下面记录了神仙做法...

    在Index中，不妨假设\(n\)为奇数，那么，均匀随机生成一个长为\(n\)​的，Alice和Bob所共有的01串，Bob生成一个bit，\(b = r_i\)，而Alice也生成一个bit，\(a = [d_H(x, r) < \frac{n}{2}]\)，经过

    计算，得到\(Pr[a = b] = \begin{cases} \frac{1}{2} - \frac{c}{\sqrt n} &(x_i = 1) \\ \frac{1}{2} + \frac{c}{\sqrt n} & (x_i = 0) \end{cases}\)​​​，重复\(m\)​遍这样的过程后，将生成的bit组合，得到两个串\(A, B\)，通过判定\(d_H(A, B)\)与\(\frac{m}{2}\)的大小，可以判断出\(x_i\)的值

    如此就把Gap-Hamming在\(t = \frac{n}{2}\)​的情况规约到了Index，对于\(\epsilon\)一般的情况，则在\(n\)的选取上动一些手脚即可

    F0的下限

    注意到\(2F_0(x, y) = d_H(x, y) + |x| + |y|\)​

    把计算F0的方法规约到计算Gap-Hamming

    Equality

    Alice得到一个（长为\(n\)的）串\(x\)，Bob得到一个串（长为\(n\)的）\(y\)，判断\(x\)和\(y\)​是否相等

    这个结果可能令人大吃一惊

    考虑生成两个公有随机串\(r_1, r_2\)​​，比较\(\langle x, r_1\rangle\)​​和\(\langle y, r_1\rangle\)​​，以及\(\langle x, r_2\rangle\)​​和\(\langle y, r_2\rangle\)，如果都相等，那么\(x, y\)将有\(\frac{3}{4}\)​​​的概率相等，因此这个问题的通信复杂度为\(O(1)\)
